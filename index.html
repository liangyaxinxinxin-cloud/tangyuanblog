工具与工程实践

「和模型一起工作」的日常技能
  • 在本职产品工作之外，出于兴趣持续自学 Cursor / Claude / Gemini / HiAgent 等 AI 工具，并尝试融入日常工作与生活。
  • 能使用 Cursor 做文档结构梳理、简单代码实验；用 Claude / Gemini 做调研、拆解复杂问题、写作与知识管理。
  • 能够为不同任务选择合适模型，并通过 角色设定、思维链 等方式提高稳定性。
  • 对 Prompt 设计、上下文管理、系统提示（System Prompt）有系统化方法，而不是一次性临时发挥。

智能体 · 工作流设计

从「一次性对话」到「可复用系统」
  • 能把业务流程拆解成 多步骤流水线，为每一步定义清晰输入 / 输出与质量标准。
  • 设计 多角色智能体（如：研究助手、评审官、编辑器）协同完成复杂任务。
  • 为智能体规划 记忆、工具访问权限、安全边界和失败兜底策略。
  • 在团队里沉淀「可复用工作流」，而不是一次性用完就丢的 Prompt。

Skills / 工具调用

让模型真正「做事」
  • 能够基于现有平台（如 Cursor / Claude / 其它 Agent 框架）配置与编排 Skills。
  • 擅长把业务系统、知识库、第三方 API 封装成 可调用工具，供模型在对话中灵活使用。
  • 理解工具调用的 幂等性、安全性与错误处理，能设计出稳定的「人机共舞」流程。
  • 善于用小型实验验证某个 Skill 是否真的节省了人力时间。

AI 产品思维

从概念到可用产品
  • 熟悉 AI 产品从 机会洞察 → 场景选择 → 方案设计 → MVP 验证 → 上线运营 的全链路。
  • 对 RAG、多轮对话、个性化助手等主流形态有清晰的优缺点判断。
  • 擅长把「技术可能性」转成 用户看得懂、愿意用 的产品价值主张。
  • 能够为 AI 功能设计 可落地的评估指标：如节省时间、错误率、满意度等。

更早之前 产品通识积累
传统互联网产品经验 → 向 AI 方向迁移
